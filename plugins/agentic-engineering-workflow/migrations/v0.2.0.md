# Migration v0.1.x → v0.2.0

## Summary

The v0.2.0 release made the setup inspection-first + consent-based and introduced the canonical "Tracker destinations" section in AGENTS.md (which `/defect` and `/idea` read at runtime). v0.1.x installs predate that section — this migration adds it, repointing slash-command destinations from hardcoded IDs to the runtime-resolved section.

This migration exists retroactively. It was added in v0.3.x once the upgrade mechanism existed; v0.1.x → v0.2.0 users who skipped this path historically would have been misdetected as PARTIAL because of the missing section.

## Changes

### agents-md-section-rename: "## Tracker — Linear" → "## Tracker destinations"

**Scope:** team-wide
**Automatable:** partial
**Action:** Rename the existing AGENTS.md section heading (if it exists with the older format like `## Tracker — Linear` or similar with concrete team/project info inside) to `## Tracker destinations`. Reshape the body into the canonical structure (Tracker type / Workspace / `/defect` destination / `/idea` destination / Planning cadence / Issue ID prefix / Workflow version).

**Before (example v0.1.x format):**
```markdown
## Tracker — Linear
- Workspace: <name> (id: <id>)
- Bug Triage project: <id>
- Ideas project: <id>
- Issue ID prefix: PROJ-NN
```

**After (v0.2.0+ format):**
```markdown
## Tracker destinations

> This section is the canonical signature that the setup has run here.
> /defect and /idea read it to know where to file.

- **Tracker type:** Linear
- **Workspace / team:** <name> (id: <id>)
- **`/defect` destination:**
  - Mode: project
  - Target: <Bug Triage project name> (id: <id>)
- **`/idea` destination:**
  - Mode: project
  - Target: <Ideas project name> (id: <id>)
- **Planning cadence:** existing-cadence
  - Detail: (carry forward whatever was set)
- **Issue ID prefix:** PROJ-NN
- **Workflow version:** v0.2.0 (last upgraded YYYY-MM-DD)
```

**Idempotency:** if `## Tracker destinations` already exists with required fields, skip. If `## Tracker` exists with old content, rename + reshape. If neither exists, the install isn't v0.1.x and this migration doesn't apply.

### agents-md-section-add: "## Docs location"

**Scope:** team-wide
**Automatable:** partial
**Action:** Add a "## Docs location" section to AGENTS.md, populated from the workspace's existing docs location. If a `docs/` folder exists, use that; otherwise ask the user where docs should live now.

**Template:**
```markdown
## Docs location

- **ADR location:** docs/decisions/ | <docs-repo>/decisions/ | <tracker-attached> | skipped
- **Strategy / PRD docs:** <path or tracker location>
- **Design source:** <path or URL>
```

### tracker-config-update: Repoint /defect and /idea to read from AGENTS.md

**Scope:** local
**Automatable:** yes
**Action:** v0.1.x installs had `/defect` and `/idea` SKILL.md files with hardcoded project IDs. v0.2.0+ skills read destinations from AGENTS.md at runtime. If the workspace has `.claude/skills/defect/SKILL.md` and `.claude/skills/idea/SKILL.md` with hardcoded IDs, replace them with the current plugin's SKILL.md content (which reads from AGENTS.md). If the user is on the plugin install (not manual), the plugin update already replaced these — no action needed.

## Notes

- This migration is required infrastructure for v0.3.0+ to recognize legacy installs as INSTALLED rather than PARTIAL. Without it, the v0.3.0 setup misdetects v0.1.x installs and offers the wrong remediation.
- Users who installed v0.1.x originally will route through this migration first, then continue to v0.3.0 (and beyond) via subsequent migration files.
