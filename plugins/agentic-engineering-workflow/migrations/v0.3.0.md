# Migration v0.2.0 → v0.3.0

## Summary

Introduces the iterative-upgrade mechanism. Future plugin versions will deliver changes via migration files in `plugins/agentic-engineering-workflow/migrations/` — the setup walks through them in version order and proposes each change for confirmation. No need to re-install from scratch.

This migration itself adds the mechanism without changing user-visible workflow content. v0.4.0+ migrations will start delivering iterative content changes.

## Changes

### agents-md-section-modify: Update "Tracker destinations" section to track workflow version

**Scope:** team-wide
**Automatable:** yes
**Action:** Ensure the AGENTS.md "Tracker destinations" section includes the "Workflow version" field. v0.2.0 already added this field, but for installs that pre-date that, add it now. Set the value to `0.3.0`.

**Template (what the section should contain):**

```markdown
- **Workflow version:** v0.3.0 (last upgraded YYYY-MM-DD)
```

**Idempotency:** if the field already exists and is set to v0.3.0 or higher, skip. If lower, update.

### memory-add: Track upgrade history locally

**Scope:** local
**Automatable:** yes
**Action:** Add a `reference_workflow_upgrades.md` memory file recording the local user's view of upgrade history. Useful for context in future sessions.

**Template:**

```markdown
---
name: workflow-upgrades
description: History of agentic-engineering-workflow upgrades applied in this workspace, from this user's perspective.
metadata:
  type: reference
---

- **Current installed version:** v0.3.0
- **Last upgrade run by me:** YYYY-MM-DD
- **Upgrade history:**
  - v0.2.0 → v0.3.0 (introduced iterative-upgrade mechanism)
```

### manual-step: Re-read setup-agentic-workflow SKILL.md mental model

**Scope:** local (informational)
**Automatable:** no
**Action:** The setup-agentic-workflow skill now supports UPGRADE mode in addition to FRESH / LOCAL SYNC / COMPLETE-PARTIAL / TEAM UPDATE / RE-INSTALL. Next time you invoke `setup-agentic-workflow` on a workspace that's behind, you'll see the upgrade flow.

No file or tracker changes needed for this item — just awareness.

## Notes

- This is a meta-migration. The infrastructure for UPGRADE mode is introduced in this version; the first real content-bearing upgrade will be the v0.4.0 migration.
- Existing v0.2.x installs will see the UPGRADE option appear when they next invoke `setup-agentic-workflow`. Picking it walks through this migration.
- If a workspace was installed pre-v0.2.0 (when the "Tracker destinations" section didn't exist yet), they'll show up as PARTIAL install state and the setup will offer COMPLETE-PARTIAL mode to add the section before this migration applies.
